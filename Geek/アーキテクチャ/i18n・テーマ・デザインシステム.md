# i18n・テーマ・デザインシステム

Geek (geek-v4) の「見た目と言語」の土台。**多言語化 (i18n)**、**ライト/ダークテーマ**、**デザイントークン (色・余白・角・サイズ・タイポ)**、**モーション/ハプティクス規約** をまとめる。React Native (Expo) の単一コードで iOS / Android / Web に同じ見た目を配信する。アクセントカラーは `#7C6AF7` (紫)。

> 出典: `CLAUDE.md` §5.5 / `lib/i18n.ts` / `lib/i18n/dictionary.ts` / `hooks/useT.ts` / `design/{tokens,typography,motion,haptics,shadows,tabbar}.ts` / `lib/theme/{palettes,themeStore}.ts` / `hooks/{useColors,useHaptic,useReducedMotion}.ts` / `stores/languageStore.ts` / `app/_layout.tsx`。

---

## 概要

3 つの独立サブシステムが `design/` と `lib/theme/` と `lib/i18n*` に分かれて同居している。

| サブシステム | 単一ソース | 購読方法 | テーマ追従 |
|---|---|---|---|
| カラー (色) | `lib/theme/palettes.ts` (`PALETTE_DARK`/`PALETTE_LIGHT`) | `useColors()` (hook) **または** `import { C }` (static) | する (2 経路) |
| 余白/角/サイズ/影 | `design/tokens.ts` (`SP`/`R`/`SIZE`/`SHADOW`) | static import | 影のみ theme 連動 (`useShadows`) |
| タイポグラフィ | `design/typography.ts` (`FONT`/`T`/`LOGO_FONT`) | static import | しない (色は別途 `textColor()`) |
| モーション | `design/motion.ts` (spring/easing/timing) | static import | reduce-motion で挙動変更 |
| ハプティクス | `design/haptics.ts` (`hap`) | `useHaptic()` / 直 `hap.tap()` | — |
| i18n (静的UI) | **2 つ並存** (下記★) | `useT()` | — |
| i18n (動的本文) | `lib/i18n.ts` `translateDynamic` | imperative | — |

要注意なのが **テーマ追従が 2 経路** (mutable な static `C` のホットスワップ + hook 購読) ある点と、**i18n に同名 `useT` が 2 系統** ある点。どちらも歴史的経緯で二重化しており、混在が地雷源 (後述)。

---

## 仕組み・設計

### 1. i18n — 静的 UI ラベル (★ 2 系統が並存)

同じ名前 `useT` の翻訳フックが **2 ファイル** に存在する。import 元で挙動が変わるので混同しない。

**系統 A: `lib/i18n.ts` (ja キー方式)**
- 辞書 `DICT: Record<string, Partial<Record<Lang, string>>>` の **キーが日本語文字列そのもの**。
- `const t = useT(); t('好きなタグ')` のように **ja 文字列を渡す**。`lang === 'ja'` なら素通し、辞書に無ければ ja をそのまま返す (安全な fallback)。
- imperative 版が 3 つ:
  - `translate(jaText, lang)` — lang を明示指定。
  - `translateStatic(jaText)` — hook を使えない場所 (store action / catch / lib 内部) 用。内部で `useLanguageStore.getState().lang` を読む。例: `stores/toastStore.ts` がトースト文言をこれでローカライズ。
  - 言語は ja/en/zh/ko/es/fr が中心 (DICT のエントリは多くがこの 6 言語)。
- 守備範囲: フィードラベル、タブ名、投稿アクション、トースト/エラー文、CW ラベル、聖地カテゴリ等。

```ts
// lib/i18n.ts — ja キーをそのまま渡す
export function useT() {
  const lang = useLanguageStore((s) => s.lang);
  return (jaText: string): string => {
    if (lang === 'ja') return jaText;
    const entry = DICT[jaText];
    if (!entry) return jaText;          // 未登録は ja のまま (壊れない)
    return entry[lang] ?? jaText;
  };
}
```

**系統 B: `hooks/useT.ts` + `lib/i18n/dictionary.ts` (dot キー方式)**
- 辞書 `STRINGS` の **キーが `common.save` のような dot 区切り namespace**。`{n}` 補間に対応。
- `const t = useT(); t('common.save')` / `t('common.minutes_ago', { n: 5 })`。
- ja/en/ko/zh/th/vi/fr/es を含む (ASEAN 展開を見据え th/vi/id も持つエントリあり)。fallback は `entry[lang] ?? entry.ja ?? key`。

```ts
// lib/i18n/dictionary.ts — {n} 補間あり
export function t(key, lang, vars = {}) {
  const entry = STRINGS[key];
  if (!entry) return key;
  let str = entry[lang] ?? entry.ja ?? key;
  for (const [k, v] of Object.entries(vars)) str = str.replaceAll(`{${k}}`, String(v));
  return str;
}
```

→ **どちらを import するかでキーの形が違う** (`'保存'` vs `'common.save'`)。CLAUDE.md §5.5 が指すのは系統 A (`lib/i18n.ts`)。新規実装でどちらを使うかは既存ファイルの import を合わせること。

### 2. i18n — 動的コンテンツ (投稿/コメント本文)

- `lib/i18n.ts` の `translateDynamic(text, targetLang)`。**MyMemory API** (無料・APIキー不要・上限あり) を `fetch`。
- 結果は module 内 `Map` キャッシュ (`${targetLang}:${text}`)。失敗時 (`!res.ok` / 例外) は **原文をそのまま返す** fail-safe。
- 🔴 **ブランド名 "Geek" を絶対に翻訳させない** 仕組みが必須:
  - `protectBrandNames(text)` で `\bGeek\b` → `__GEEKBRAND__` に置換 → API 通過 → `restoreBrandNames(out)` で復元 (復元は case-insensitive)。
  - 理由: MyMemory に "Geek" を渡すと「オタク」「긱」「极客」等の一般名詞に化ける。`__GEEKBRAND__` は underscore 囲み uppercase token で API がトークン分割翻訳しにくい。
  - マッチは `\bGeek\b` のみ (小文字 `geek` は一般名詞として翻訳 OK、`GEEK`/全角は保留)。
  - 回帰テスト: `tests/unit/i18nBrand.test.ts`。

### 3. i18n — Web のブラウザ翻訳連動 (★ 実は dict より主役)

`app/_layout.tsx` の effect が **Web ではブラウザ翻訳機能を主たる多言語化手段** にしている (CLAUDE.md §5.5)。

- `document.documentElement.lang = lang` を更新 → Chrome/Edge/Safari の「このページを翻訳」発火 + ARIA 読み上げ言語 + CSS `:lang()` が効く。
- ⚠️ `scripts/fix-html.mjs` が `dist/index.html` に `<html translate="no">` + `<meta name="google" content="notranslate">` を注入してブラウザ翻訳をブロックしている。そのため:
  - `lang === 'ja'` → ブロック維持 (日本語ユーザーに勝手な訳が入らない既存挙動)。
  - `lang !== 'ja'` → effect が `translate='yes'` に上書き + notranslate meta を除去して翻訳プロンプトを発火。
- ★ コメントによれば「旧実装は `useT()` を定義したが UI 内に使用箇所 0 だったため languageStore を変えても文字列が翻訳されないバグだった」→ Web はブラウザ翻訳に寄せ、Native は `useT()`/`translateDynamic` を順次拡充する方針。**= Web の UI ラベル翻訳は dict ではなくブラウザ翻訳依存**という実態を理解しておくこと。

### 4. languageStore (Zustand)

`stores/languageStore.ts`。
- `Lang = 'ja' | 'en' | 'zh' | 'ko' | 'es' | 'fr' | 'th' | 'vi' | 'id'` (9 言語)。`LANG_OPTIONS` に native 名+flag。
- 永続化は `AsyncStorage` (key `geek:lang`、`{ lang, autoTranslate }` を JSON)。失敗は `swallow('store.language.*')`。
- `autoTranslate` (投稿/コメントの自動翻訳 on/off) を **lang とは独立** に保持。
- ★ **2026-05-25 改修**: `setLang('en')` で `autoTranslate=true` を自動セットしていた旧仕様を撤廃。「言語を変えたら勝手に日本語投稿が翻訳された」production 事故の対策。今は `setLang` は lang のみ更新、`setAutoTranslate` で別途切替。設定→言語画面 (`app/settings/language.tsx`) に独立 toggle。

### 5. テーマ — palette とモード解決

**色の単一ソースは `lib/theme/palettes.ts`** (`PALETTE_DARK` / `PALETTE_LIGHT`、両者は同じ `ColorPalette` shape)。

- `ColorPalette` は ~50 キー: 背景 `bg`〜`bg5`/`surfaceHi`、`glass*`、`text`〜`text4`、`border*`/`divider`、`accent*`、状態色 `green/amber/red/pink/blue` + `*Bg`、`block*`、`liked*`、リレーション色 `related/sameGroup/sameGenre` + `*Bg`/`*Border`、`trustLow/Mid/Hi`、`scrim*`。
- **Light palette の設計指針** (コメント明記):
  - `bg` は純白 `#fff` ではなく `#ffffff`/`#fafafc` 系 (純白は眩しい)、`text` は `#1a1a1a` (純黒はコントラスト過剰)。
  - accent は light 専用で**「自然な青」`#3E6DA3`** (彩度控えめ、白で 5.3:1)。鮮やかな blue/cyan は白で浮くため不採用。
  - 状態色 (red/amber/green) は WCAG AA を満たす濃いめ。
  - `glass` は黒 base に変更 (白背景に `rgba(255,255,255,*)` は見えない)。
  - `border` は `#c4c4c9` (旧 `#e4e4e7` は薄すぎて枠が見えなかった)。

**モード store: `lib/theme/themeStore.ts`** (Zustand、provider 不要)。
- `ThemeMode = 'system' | 'light' | 'dark'`、永続キー `geek:theme:mode` (同期 storage)。
- ★ **既定は `'dark'`** (`'system'` 既定にしない)。理由: 既存ユーザーを「いきなりライト化」させない / iOS light 端末で突然真っ白になるのを防ぐ。ユーザーが明示的に `system` を選んだ時だけ OS 連動。
- `useResolvedTheme()` が `'light' | 'dark'` に解決。`mode === 'system'` のときだけ RN `useColorScheme()` を読む (常に呼ぶことで system 設定変更時に自動 re-render)。

### 6. テーマ追従の 2 経路 (★ ここが最重要かつ事故源)

色を使う方法が **hook 経由** と **static `C` 経由** の 2 つあり、後者が特殊。

**経路 A: hook 購読 (`hooks/useColors.ts`)** — 新規/移行対象が使う正攻法。
- `useColors()` → `useResolvedTheme()` を購読し `useMemo` で palette を返す (同一テーマ render では同一参照を返し StyleSheet 比較を壊さない)。
- 兄弟: `useGradients()` (`GRAD_DARK`/`GRAD_LIGHT`)、`useShadows()` (`SHADOW_DARK`/`SHADOW_LIGHT`、light は影が薄め)、`useTheme()` (`{ C, GRAD, SHADOW, isDark }` を 1 発取得)。

**経路 B: static `import { C } from '../design/tokens'` (mutable ホットスワップ)** — 既存 193+ ファイルが直参照。
- `design/tokens.ts` の `C` は **mutable な単一オブジェクト参照**。dark 既定値を持つ `_C` を `export const C` が指す。
- テーマ切替時に `applyThemeC(palette)` が `Object.assign(_C, palette)` で **中身だけ破壊的に書き換える** (参照は維持)。→ 全 importer が同じ object を持っているので一斉に追従。
- ただし**値書換だけでは React は再 render しない**ので、`app/_layout.tsx` 側で `<View key={theme}>` 相当の **key remount** を併用して tree を作り直す。
- `GRAD` も同様に `applyThemeGRAD()` で実体配列だけ差替 (型は `as const` tuple 固定のまま、`unknown` 経由 cast で書換)。light は brand grad が「自然な青」、dark は紫。
- `themeStore.ts` の `syncStaticPaletteWithTheme(theme)` が `applyThemeC` + `applyThemeGRAD` をまとめて呼ぶ。`_layout.tsx` は `syncStaticPaletteIfChanged(resolvedTheme)` を **render の commit 段階で同期実行** (useEffect だと初回 paint で一瞬古い色がフラッシュするため)。
- `isLightActive()` ヘルパ: `_C.bg !== _DARK.bg` で現在 light かを hook 無しで判定 (web の CSS gradient だけ theme 分岐したい箇所用)。

> ⚠️ なぜ palette を `tokens.ts` に二重定義しているか: `tokens.ts` が `palettes.ts` を import すると**循環参照**になるため、`_DARK` を `tokens.ts` に直書きし「`palettes.ts` と shape/値を手で完全一致」させている。片方だけ色を足すと不整合になる。

### 7. デザイントークン (`design/tokens.ts`)

色以外のスケール。すべて `as const`。

- **`SP` (余白)**: `0=0, 1=4, 2=8, 3=12, 4=16, 5=20, 6=24, 7=28, 8=32, 10=40, 12=48, 16=64, 20=80, 24=96` (4px グリッド)。
- **`R` (角丸)**: `none=0, sm=6, md=10, lg=14, xl=20, 2xl=28, 3xl=36, full=9999`。
- **`SIZE`**: タッチ/アバター/アイコン/バー寸法。`touch=44`(最小タップ), `touchLarge=56`, `avatarSm/Md/Lg/Xl=28/36/56/96`, `iconSm/Md/Lg/Xl=16/20/24/28`, `tabBar=64`, `tabIcon=26`, `fab=56`, `input=48`, `button{Sm,Md,Lg}=36/48/56` 等。
- **`SHADOW` (tokens 版)**: `none/xs/sm/md/card/cardPress/accentGlow/glow`。tokens.ts のは Platform 非分岐の新版で「新コードはこちら推奨」。
- **`design/shadows.ts`** は別の Platform 分岐 (`Platform.select`) 版 `SHADOW` を re-export (`fab/press/pill` 等を追加で持つ)。両者は同期しているが**同名 export なので import 元注意**。theme 連動が必要なら `useShadows()`。
- **`design/tabbar.ts`** `TABBAR`: `height=64, iconSize=26, iconStroke=2.2, indicatorH=3, indicatorW=28, bgBlur=24, fabSize=56, fabOffset=-8, labelGap=2`。

### 8. タイポグラフィ (`design/typography.ts`)

- **`FONT`** (font family、★ weight を絞ってバンドル軽量化): `display/display2='Syne_700Bold'`, `ui/uiBold='Inter_600SemiBold'`, `ui400='Inter_400Regular'`, `jp='NotoSansJP_400Regular'`, `jpM/jpB='NotoSansJP_700Bold'`。`display2→display`、`uiBold→ui`、`jpM→jpB` に**集約済** (API は維持、参照先のみ統合して weight 数を削減 — パフォーマンス対策)。
- **`T`** (text style プリセット): `hero(40)/display(34)/h1(28)/h2(22)/h3(18)/h4(16)/body(15)/bodyM/bodyB/small(13)/caption(11)/num/numLg/mono/button{Lg,Md,Sm}`。見出しは負の `letterSpacing`。
- **`textColor(kind)`**: `primary→C.text / secondary→C.text2 / tertiary→C.text3 / disabled→C.text4`。色は static `C` 参照なのでテーマ追従する。
- **ロゴ (`LOGO_FONT`)**: Apple system stack。iOS=`System`, web=`-apple-system, BlinkMacSystemFont, "SF Pro Display", ...`, Android=`Inter_700Bold`。`LOGO_FONT_WEIGHT='700'`。
  - ⚠️ 注: `typography.ts` の `LOGO_FONT_WEIGHT` は **'700'**。ただし起動スプラッシュ/イントロの確定仕様 (CLAUDE.md §0) は **weight 800** を要求。両者は文脈が別 (スプラッシュは `web-postbuild.mjs`/`IntroAnimation.tsx` の固定値) なので**混同して片方に揃えない**。詳細は [[地雷・落とし穴 総覧]]。
- **ブランドグラデ文字 `geekGradientFill()`**: 単一ソース `GEEK_GRADIENT_CSS = 'linear-gradient(120deg, #7C6AF7 0%, #B98CFF 48%, #E891C7 100%)'`。
  - web: CSS `background-clip:text` (`color:transparent` + `backgroundImage` + `backgroundClip` + `WebkitBackgroundClip` + `WebkitTextFillColor` の 5 プロパティセット必須。react-native-web 0.19.13 が全 prop を DOM へ通すことをソース確認済)。
  - native: RN Text はグラデ文字不可 → 単色フォールバック `#B98CFF`。
  - ★ stop 値は `scripts/web-postbuild.mjs` の起動スプラッシュ `.gk-word` と一致させること (seam 防止)。

### 9. モーション規約 (`design/motion.ts`)

Reanimated 3 + Moti。値は Apple HIG / Apple Photos / Reddit Android の press feedback を参考にチューニング。

- **Spring** (`WithSpringConfig`):
  - `SPRING_SNAPPY` `{damping:18, stiffness:300, mass:0.6}` — タップ press-feedback 標準 (PressableScale)。
  - `SPRING_SOFT` `{22,200,0.8}` — overlay/sheet/focus ring など重い物。
  - `SPRING_BOUNCY` `{12,280,0.7}` — ハート/スタンプの「効果」。少し弾む。
  - `SPRING_TIGHT` `{18,320,0.7}` — segmented control / toggle。
  - `SPRING_GENTLE` `{22,180,1}` — progress bar など連続変化。
  - `SPRING_SNAP` = `SPRING_SNAPPY` の legacy alias (残置)。
- **Easing**: `EASE_OUT=bezier(.22,1,.36,1)`, `EASE_IN_OUT=bezier(.65,0,.35,1)`, `EASE_OUT_BACK=bezier(.34,1.56,.64,1)`, `EASE_OUT_QUART=bezier(.165,.84,.44,1)`。
- **Timing**: `TIMING_FAST(120ms, quart-out)` micro-interaction / `TIMING_NORM(220ms, ease-out)` 標準 (`TIMING_NORMAL` は alias) / `TIMING_SLOW(380ms)` modal・sheet。
- **コンポーネント定数**: `PRESS_SCALE=0.96`, `PRESS_SCALE_TIGHT=0.94`, `FAB_SCALE=0.92`, `HEART_SCALE_KEYFRAMES=[0,1.3,1.0,1.15,1.0,0.9,0]`, `SHIMMER_DURATION=900`, `SCREEN_TRANSITION=280`, `MODAL_TRANSITION=320`, `TOAST_DURATION=2400`。
- **`clampHandoff(vNorm, toValue)`** (worklet): ジェスチャ解放時の指速度を `withSpring` の `velocity` に渡す前に整形 — 目標方向と同符号成分のみ採用 (逆フリック overshoot 防止) + 絶対値 `MAX=8` cap (過大初速でバネが暴れるのを防ぐ)。HomeDrawer / feed の open・close で共用。
- **reduce-motion**: `hooks/useReducedMotion()` = `useSettingsStore((s) => s.reduceMotion)` (アプリ内設定。OS 設定とは別の自前フラグ)。「動きは本当に必要な時だけ」が HYPOTHESIS_LOG 原則。起動イントロの reduce-motion 挙動は CLAUDE.md §0 に確定仕様あり (`ReduceMotion.Never` で fade を必ず動かす等)。

### 10. ハプティクス規約 (`design/haptics.ts`)

- `hap` オブジェクト (expo-haptics ラッパ、iOS/Android のみ作動、web/失敗は no-op で `try/catch` 握り潰し):
  - `tap`=Impact Light / `confirm`=Impact Medium / `pop`=Impact Heavy / `select`=Selection / `success`/`warn`/`error`=Notification 各種。
- `hooks/useHaptic(kind='tap')` → `useCallback(() => hap[kind](), [kind])`。
- ★ タップ感の事故対策 (CLAUDE.md §11): PressableScale は `delayPressIn=0` + **haptic は `onPressIn` に紐付け** (`onPress` だと体感が遅れる) + `hitSlop:8`。

## 注意点・地雷

- **★ `useT` が 2 つある**: `lib/i18n.ts` (ja キー `t('保存')`) と `hooks/useT.ts`+`lib/i18n/dictionary.ts` (dot キー `t('common.save')`、`{n}` 補間)。**import 元でキーの形が違う**。既存ファイルに合わせる。CLAUDE.md §5.5 が指すのは前者。
- **★ Web の UI ラベル翻訳は dict ではなくブラウザ翻訳依存**: `_layout.tsx` のコメント通り「`useT()` の UI 内使用箇所が 0 でバグっていた」経緯があり、Web は `document.documentElement.lang` + `fix-html.mjs` の notranslate 動的解除でブラウザ翻訳に寄せている。Native だけ dict/`translateDynamic` を拡充中。**「dict にキーを足せば Web の UI が翻訳される」とは限らない**。
- 🔴 **"Geek" は翻訳禁止**: `translateDynamic` は必ず `protectBrandNames`→API→`restoreBrandNames`。test `i18nBrand.test.ts`。外すと「オタク/긱/极客」化。
- **★ palette が 2 箇所に二重定義**: `lib/theme/palettes.ts` (`PALETTE_DARK`) と `design/tokens.ts` (`_DARK`) は循環 import 回避のため手で同期。**色キーを足す時は両方** + light 値も。
- **★ static `C` は mutable ホットスワップ**: テーマ切替は `applyThemeC`/`applyThemeGRAD` が `Object.assign`/実体差替で**破壊的更新**し、**key remount で再 render を起こす**。値書換だけでは再 render しない。`_layout.tsx` の `syncStaticPaletteIfChanged` は **commit 段階で同期実行** (useEffect だと初回フラッシュ)。新規 component は素直に `useColors()` を使う方が安全。
- **★ theme 既定は `dark`**: `'system'` 既定にすると iOS light 端末で突然真っ白。`useResolvedTheme()` で解決し、`system` の時だけ OS 連動。
- **`SHADOW` が 2 系統**: `design/tokens.ts` (Platform 非分岐・新) と `design/shadows.ts` (Platform.select・`fab/press/pill` あり) で同名 export。theme 連動が要るなら `useShadows()` (`SHADOW_DARK`/`SHADOW_LIGHT`)。light の影色は黒ではなく slate-400 `#94a3b8` (白に黒影は「中途半端に黒い」気持ち悪さが出るため、2026-05-31 改修)。
- **ロゴ weight の文脈差**: `typography.ts` の `LOGO_FONT_WEIGHT='700'` と、起動スプラッシュ/イントロ確定仕様の weight 800 (CLAUDE.md §0) は**別物**。揃えようとしない。
- **light の accent は青、dark は紫**: light は `#3E6DA3` (自然な青)、dark は `#7C6AF7` (紫)。`GRAD` も theme で差替 (`applyThemeGRAD`)。「青UI に紫グラデが混ざる」を解消した経緯があるので、片方だけ色を直さない。
- **reduce-motion は OS でなくアプリ内設定**: `useReducedMotion()` は `settingsStore.reduceMotion` を読む自前フラグ。OS の Reduce Motion とは独立 (起動イントロだけは別途 `ReduceMotion.Never` 等を使う、CLAUDE.md §0)。
- **言語変更で自動翻訳しない (2026-05-25〜)**: `setLang` は lang のみ更新。`autoTranslate` は別 toggle。旧仕様 (lang 変更で勝手に翻訳 on) の事故を踏まないこと。
- **font weight は絞ってある**: `FONT` で `display2/uiBold/jpM` を実体統合済 (バンドル/メモリ削減)。新規で `Syne_600`/`NotoSansJP_500` 等を足すと weight が増えて重くなる → [[パフォーマンス最適化]] (フォント 26MB の hotspot あり)。
