# リキッドタブインジケーター完全ガイド (React Native + Reanimated 3)

> 作成日: 2026-06-12
> 対象: Geek v4 (Expo Router + Reanimated 3 + React Native Web)
> テーマ: Slack / Reddit / Threads 級の「スーッと動く・ピル型・モーフィング」タブインジケーターの実装

---

## エグゼクティブサマリー

Slack / Reddit 級の滑らかなスライドピル・インジケーターを React Native (Expo + Reanimated 3) で作るための核は次の4点に集約される:

1. **物理ベースの spring を使う**(ease ではなく)。spring だけがジェスチャ速度を引き継げ、途中で目的地が変わったときも違和感なくリターゲットできる。Apple の WWDC23 ガイダンス・Reanimated の `withSpring` の両方が同じ理屈で springs を推奨している。
2. **`translateX` / `scaleX` だけを動かす**。`width` / `left` のようなレイアウト系プロパティを毎フレーム触るとレイアウト再計算が走り、60fps が崩れる(Reanimated 公式の performance guide)。
3. **インジケーターのドライバは単一 `useSharedValue`**(focused index か scroll position)で、`useAnimatedStyle` 内で `interpolate()` してタブ幅 / 位置の配列を読む。タブ寸法は `onLayout` で `useSharedValue<Array>.modify()` に書き込む(`itemDimensions.value[i] = x` は Reanimated 3.7.0+ で silent fail するため絶対に使わない)。
4. **Shared Element Transitions は使わない**。tab navigator では「アニメが走らない」と公式に明記されており、Web 未対応・experimental flag 必須。framer-motion の `layoutId` は Web ビルドのみで使い、ネイティブは手書きの shared-value パターンに統一するのが安全。

iOS 26 の「Liquid Glass」タブバーが OS レベルでピル+モーフ+squash bounce を提供したことで、ユーザーの「滑らかさ期待値」はさらに上がった。サードパーティが OS レベルの水滴マージ(`UIGlassContainerEffect`)を完全再現するのは現実的でないが、Geek アプリでは **`bounce ≈ 0.2` 相当 (Reanimated 換算で `dampingRatio ≈ 0.75`, `duration: 350ms`)** + `translateX` で十分「Slack 並み」に到達できる。

| 通称 | 正式名 | 出典 |

## 2. プラットフォーム仕様

### 2.1 Apple HIG / iOS 26 Liquid Glass (Confidence: high)

- **タブバーが浮く**: iOS 26 は `UITabBarController` を Liquid Glass に「持ち上げ」、コンテンツ上に float させる(WWDC25 session 284)。
- **カプセル型 + sliding indicator**: 選択カプセルが OS 標準の active indicator。ネイティブの slide アニメ付き。
- **水滴マージ**: `UIGlassContainerEffect.spacing` が「ガラス同士が近接すると水滴のように1つの形にマージ」する閾値を制御。
- **タップで scale + bounce**: `glassEffect(isInteractive: true)` で物理 spring 駆動の scale & bounce(shimmer も含む)。Apple 表現で「entirely new physics-informed paradigm」。
- **重要な制約**: フルの「Liquid blob ドラッグ歪み」は **private API**(サードパーティでは完全再現不可)。

### 2.2 Material 3 (Confidence: medium-high)

- **Active indicator** が正式名(ピル状の背景)。
- ナビゲーションバーアイテムのタッチ領域は **48dp 最小**(target size)。
- **State layer**(hover / pressed / focused の半透明オーバーレイ)が active indicator と別レイヤとして重なる。
- Material Symbols との関係: アイコンは `filled` / `outlined` の duotone 切替で選択状態を補強(indicator と二段構え)。

| アプリ | 形状 | 移動 | squash | 推定 timing |

## 4. Reanimated 3 実装パターン (Confidence: high)

### 4.1 推奨アーキテクチャ

```tsx
// 1. ドライバ: 単一 SharedValue
const activeIndex = useSharedValue(0);

// 2. タブ寸法: SharedValue 配列 (modify() で書き込む)
const tabLayouts = useSharedValue<{ x: number; w: number }[]>([]);

// 3. 各タブで onLayout
const onTabLayout = (i: number) => (e: LayoutChangeEvent) => {
  const { x, width: w } = e.nativeEvent.layout;
  tabLayouts.modify((arr) => {
    'worklet';
    arr[i] = { x, w };
    return arr;
  });
};

// 4. インジケーターは translateX + scaleX
const indicatorStyle = useAnimatedStyle(() => {
  const layouts = tabLayouts.value;
  if (!layouts.length || !layouts[activeIndex.value]) return { opacity: 0 };
  const { x, w } = layouts[activeIndex.value];
  return {
    opacity: 1,
    transform: [
      { translateX: x },
      { scaleX: w / BASE_WIDTH }, // BASE_WIDTH は固定値
    ],
  };
});

// 5. タップ時に spring で更新
const onPressTab = (i: number) => {
  activeIndex.value = withSpring(i, GEEK_SPRING_CONFIG);
};
```

### 4.2 絶対にやってはいけないアンチパターン

| アンチパターン | 何が起きる | 正解 |
| --- | --- | --- |
| `tabLayouts.value[i] = x` で代入 | **Reanimated 3.7.0+ で silent fail**(配列が永遠に空) | `.modify((arr) => { 'worklet'; arr[i] = ...; return arr; })` |
| JS thread で `sv.value` を read | JS スレッドが UI スレッド同期完了までブロック(>500ms 出ることも) | worklet 内のみで read |
| `width` / `left` をアニメ | 毎フレーム layout 再計算 → ドロップフレーム | `transform: translateX + scaleX` |
| Shared Element Transitions を tab で使う | 公式に「animation may not run」 | 手書きの shared-value indicator |
| `initialLayout` を渡さない | 初回 render で jitter | `initialLayout={{ width: layout.width }}` |

### 4.3 spring config 設計

**Reanimated 3 の `withSpring` には2系統の API**(混在不可):

A. **物理ベース**: `stiffness` (default 900) + `damping` (default 120)
B. **知覚ベース**: `duration` (default 550ms, 実時間は 1.5x) + `dampingRatio` (default 1, <1 で underdamped/bouncy, >1 で overdamped)

Apple WWDC23 の `spring(duration:bounce:)` API では:
- `bounce = 0` = critically damped(「迷ったらこれ」の汎用デフォルト)
- `bounce > 0.4` は **UI には強すぎ**(Apple 自身が明確に警告)
- `bounce` 範囲は -1.0〜1.0

## 5. 「Geek アプリで採用する spring 値」推奨

### 5.1 早見表

| プリセット | duration | dampingRatio | 用途 | Apple bounce 換算 |
| --- | --- | --- | --- | --- |
| `GEEK_SPRING_INDICATOR` | **350ms** | **0.75** | ピル移動(本命) | ~0.2 |
| `GEEK_SPRING_SUBTLE` | 250ms | 1.0 | iOS 風控えめ | 0 |
| `GEEK_SPRING_PLAYFUL` | 400ms | 0.6 | スタンプ反応など | ~0.35 |
| ❌ NG | - | <0.5 | ガタガタ感 / UI には強すぎ(Apple警告) | >0.4 |

### 5.2 採用コード

```ts
// geek-v4/lib/motion/springs.ts
export const GEEK_SPRING_INDICATOR = {
  duration: 350,
  dampingRatio: 0.75,
} as const;

export const GEEK_SPRING_SUBTLE = {
  duration: 250,
  dampingRatio: 1.0,
} as const;

export const GEEK_SPRING_PLAYFUL = {
  duration: 400,
  dampingRatio: 0.6,
} as const;
```

### 5.3 採用根拠

- **Apple WWDC23 の bounce=0** は最も汎用的だが、Geek のキャラクター(SNS的 / 若年層)を踏まえ「ほんのり弾む」 bounce ~ 0.2 相当を default に。
- Apple 警告ライン `bounce > 0.4` は越えない(UI には exaggerated すぎる)。
- 350ms は Slack/Reddit 体感と一致するレンジ。

## 6. パフォーマンス・60fps 維持の鉄則

| ✅ Do | ❌ Don't |
| --- | --- |
| `translateX` / `scaleX` / `opacity` / `backgroundColor` だけアニメ | `width` / `left` / `top` / `margin` / `padding` をアニメ |
| `useSharedValue` を worklet 内で読む | JS thread から `sv.value` を読む |
| `tabLayouts.modify()` で配列書き換え | `tabLayouts.value[i] = x` |
| `initialLayout` を必ず渡す | render 中の jitter を放置 |
| spring の `dampingRatio >= 0.5` | <0.5 でガタつかせる |

## 7. アクセシビリティ

### 7.1 prefers-reduced-motion 検出

```ts
import { AccessibilityInfo } from 'react-native';

// 初期取得
const reduce = await AccessibilityInfo.isReduceMotionEnabled();

// 動的購読
const sub = AccessibilityInfo.addEventListener(
  'reduceMotionChanged',
  (enabled) => setReducedMotion(enabled),
);
```

- **iOS**: `UIAccessibility.isReduceMotionEnabled` 直結。
- **Android**: `Settings.Global.TRANSITION_ANIMATION_SCALE == 0.0` で判定(facebook/react-native PR #23839)。
- **Web (RNW)**: 歴史的に欠落していたため、`window.matchMedia('(prefers-reduced-motion: reduce)')` でフォールバック実装が必要。

### 7.2 縮退仕様(reduce motion on 時)

| 項目 | 通常 | reduce motion |
| --- | --- | --- |
| インジケーター移動 | spring 350ms | 即座にジャンプ (or 100ms fade) |
| squash (scaleX) | 有 | 無効化 |
| Lottie / 装飾アニメ | 有 | 無効化 |

### 7.3 VoiceOver / TalkBack

- タブには `accessibilityRole="tab"` + `accessibilityState={{ selected: isActive }}`。
- インジケーター自体は `accessibilityElementsHidden` / `importantForAccessibility="no-hide-descendants"` で読み上げ対象から除外。

### 7.4 大きな文字 (Dynamic Type)

- ピル幅を **固定にしない**。テキスト幅+padding で算出 → `onLayout` で再測定 → `modify()` で SharedValue を更新する。これで Dynamic Type 拡大時もピルが追従。

## 8. framer-motion との比較 / Web ビルド戦略

| 観点 | framer-motion (Web) | Reanimated 3 (Native) |
| --- | --- | --- |
| Shared layout 抽象 | `layoutId="indicator"` 1行で完結 | 手書き(SharedValue + onLayout) |
| 物理モデル | type:"spring" + stiffness/damping | withSpring (同等) |
| 速度引き継ぎ | 自動 | 自動 (gesture 連動時) |
| Web 対応 | ◯ (本職) | △ (SET は未対応 / 手書きパターンは可) |

**Geek の Web ビルド方針**:

```tsx
// Platform で分岐(同一コンポーネント、異ロジック)
if (Platform.OS === 'web') {
  // framer-motion <motion.div layoutId="tab-indicator" />
} else {
  // Reanimated 3 手書き SharedValue パターン
}
```

→ ネイティブと web で見た目は揃え、内部実装だけ分ける。これが SET の web 非対応 + tab navigator 非対応の二重制約に対する **最もコスパの良い回避策**。

## 11. オープン課題

- Slack / Reddit / Threads 各社の **正確な spring 係数**(stiffness/damping/duration)は内部実装で非公開。観察ベース推定の精度向上には実機 60fps 録画 + 微分解析が必要。
- iOS 26 Liquid Glass の「水滴マージ」を SwiftUI/UIKit 側で `UIGlassContainerEffect` 経由で取り入れた場合、React Native ブリッジ経由でどこまで露出できるか。expo modules で wrap する余地。
- React Native Web の `prefers-reduced-motion` 検出を `AccessibilityInfo` に統合する PR の動向。
- Reanimated 4.x の Fabric 完全対応後、SET が tab navigator でも動くようになる時期(2026 末ロードマップ未確認)。
