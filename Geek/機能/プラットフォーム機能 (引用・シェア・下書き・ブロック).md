# プラットフォーム機能 (引用・シェア・下書き・ブロック)

> Geek(geek-v4)の「YouTube/Instagram/X/Reddit にあって geek に無かったもの」として 2026-06-09 に一括実装された 8 機能群。tsc 0 errors / ESLint clean、UI 統合も `AnonPostCard` + `app/post/create.tsx` 済み。
> 出典: memory `project_geek_platform_features.md` + 実コード(下記ファイルパス)。

## 概要

## 仕組み・設計

### 1. 引用投稿 (Quote Post)

- **DB** (`migration 0142_quote_posts.sql`): `posts.quote_post_id uuid references posts(id) on delete set null` の**自己参照 FK**。引用投稿は専用テーブルでなく**通常の posts 行**で表現し、`quote_post_id` が NULL でない行が引用投稿。partial index `idx_posts_quote_post_id (where quote_post_id is not null)`。引用カウント用 view `post_quote_counts` も定義(オプション)。
  - 引用元 `on delete set null` なので、引用元が消えると `quote_post_id=null` になる(= 引用関係だけ消える、引用投稿自体は残る)。
- **取得** (`lib/api/quotePosts.ts`):
  - `fetchQuotedPost(postId)` → `QuotedPostPreview | null`。削除済/存在しないと null。
  - **FlashList リサイクル対策**: モジュールレベル `_quoteCache: Map` + `_quoteInflight: Map` で **セッション内重複フェッチを完全排除**。FlashList はセルをリサイクルするため同 postId のローダが unmount→remount を繰り返しネットワークが再発する。in-flight dedup は「同 postId の fetch 進行中なら同じ Promise を返す」。`withApiTimeout(..., 8000)` + `swallow` で握り潰し。
  - select するのは `id,content,title,tag_names,created_at` のみ(プレビューに必要な最小)。
- **作成** (`createQuotePost(opts)`): `posts` へ `{ content, quote_post_id, tag_names, is_anonymous }` を insert。**mutation はリトライしない**(副作用あり, → §5.1 規約)。失敗時 null。
- **表示** (`components/post/QuotePostMini.tsx`): X/Twitter 風の丸角ボーダーボックス。
  - `post={null}` で「元の投稿は削除されました」プレースホルダ(italic, accessibilityLabel 付き)。
  - title(BBS スレタイ)は太字 1 行、content は `numberOfLines=2`、tag は先頭 `MAX_TAG_CHIPS=2` 件をチップ表示。
  - 背景 `C.bg3`(カード bg より 1 段暗い)で視覚的に区別。tag chip text は `C.text2`(WCAG AA: dark 7.72:1 / light 7.73:1)。
  - `onPress` 時は `e.stopPropagation()` で親カードのタップに伝播させない。`memo` 済。
- **UI 統合** (`components/feed/AnonPostCard.tsx`):
  - `QuotePostMiniLoaderInner`(L380) が `useQuery({ queryFn: () => fetchQuotedPost(postId) })` で引用元を取得 → `QuotePostMini` に渡す(L1158 で `post.quote_post_id` があれば描画)。
  - タップで `handleOpenQuoteDetail` → 投稿詳細へ。

### 2. クロスポスト (Cross-post)

- **DB**: 既存の `post_communities` ジャンクションテーブルを操作(専用 migration なし)。Reddit 的「同一投稿を複数コミュニティへ転載」。
- **API** (`lib/api/crossPost.ts`):
  - `crossPostToCommunity(postId, communityId)` → `post_communities` へ insert。**一意制約違反 (code `23505`) = 既に存在 → false**(エラーにしない)。
  - `getCrossPostCommunities(postId)` → PostgREST ネスト select `community_id, communities(id,name,icon_emoji)`。`communities` は object で返るので map で平坦化。エラー時は `[]`。
  - `removeCrossPost(postId, communityId)` → delete。**RLS により投稿所有者でなければ無視される**(明示的な権限チェックはクライアントに無い)。
- **★ 未完**: memory「残タスク」に **クロスポスト選択 UI(コミュニティ選択シート)** が未実装と明記。API は揃っているが呼び出す画面が無い。

### 3. シェア + 埋め込み (`lib/utils/sharePost.ts`)

- `BASE_URL = 'https://geek-app.netlify.app'`。投稿 URL は `${BASE_URL}/post/${encodeURIComponent(id)}`。
  - ⚠️ **CLAUDE.md / 他ノートでは Web 本番が `geekboard.netlify.app`**。ここの `geek-app.netlify.app` と食い違う可能性(リンク先ドメインのズレ → 共有リンクが 404 になる潜在地雷)。要確認。
- `buildShareText(post)`: タイトル or 本文を 100 字で切って `…`、tags を `#tag` 連結、`Geek で見る: ...` 形式。
- **API 一覧**:
  - `sharePost(post)`: web は `navigator.share`(Web Share API, 未対応なら `Linking.openURL`)、native は `Share.share`。Android は `message\nurl`、iOS は `url` フィールド。**AbortError / NotAllowedError / "User did not share" / "Share was cancelled" はキャンセル扱いで握り潰す**(それ以外は throw)。
  - `shareToX(post)`: `https://twitter.com/intent/tweet?text=...&url=...` を `Linking`。開けなければ throw。
  - `shareToLINE(post)`: `line://msg/text/...`(アプリ)→ 不可なら `https://social-plugins.line.me/lineit/share`(Web フォールバック)。
  - `copyPostLink(post)`: `@react-native-clipboard/clipboard` を try-require。**未導入環境では `Clipboard=null` → false**。AnonPostCard 側もこれを見越してフォールバック処理あり(L729 コメント)。
  - `getEmbedCode(post)`: `<iframe src="${BASE_URL}/embed/post/${id}" width="100%" height="400" ... loading="lazy">` を返す。`getEmbedPreviewUrl(postId)` も export。
- **UI 統合** (`AnonPostCard.tsx` L712–745): シェアボタン → シート(X 共有 / リンクコピー / 埋め込みコードコピー)。

### 4. 下書き (`stores/draftStore.ts` + migration 0144)

- **設計方針**: ローカル(MMKV/localStorage)先行保存 + Supabase RPC fire-and-forget 同期。Zustand。`MAX_DRAFTS=20`、storage key `geek:drafts:v1`。
  - **ローカル保存を先行 → RPC は非同期バックグラウンド**(`syncUpsertToServer` / `syncDeleteFromServer`、いずれも `swallow` 握り潰し)。
  - サーバー割当 UUID は `serverId` に保持し次回 upsert に渡す。サーバー RPC: `upsert_post_draft` / `delete_post_draft` / `get_my_drafts`(30 日 TTL, migration 0144)。
  - `loadDrafts()`: cold start で `getJson` 同期復元(await なし即 hydrate)→ バックグラウンドで `get_my_drafts` をマージ(同 `serverId` は `updatedAt` 新しい方を採用、ローカル専用 `serverId` 無しは保持)→ `updatedAt` 降順で `MAX_DRAFTS` cap。`hydrated` フラグで多重呼び出し防止。
  - `saveDraft` / `updateDraft` / `deleteDraft` / `getDraft` / `clearAllDrafts`(ログアウト時ローカルのみ消去, サーバー削除しない)。
  - **Zustand selector で subscribe 必須**(コメントに §5.4 destructure 禁止と明記, → [[State管理 (Zustand・React Query)]])。
- **UI 統合** (`app/post/create.tsx`):
  - `DRAFT_DEBOUNCE_MS`(memory では **1.5s**)で自動保存。「**下書き保存済み**」インジケーターをフェードイン→保持→フェードアウト(L1032–1044)。
  - `?draftId=xxx` クエリで「下書き一覧」から再開(mount 1 回, L179–207)。**編集モード (`editIdRef`) は下書きへ書き戻さない**(L284)。
  - server-synced の `loadDrafts` を mount 時 1 回呼ぶ(L221–227)。
- **★ 未完**: memory「残タスク」に **下書き一覧 UI(draft list screen)** が残課題と記載(`?draftId=` で開く側はあるが一覧表示の独立画面が未整備)。

### 5. スパム検出 (`lib/safety/spamDetection.ts` + migration 0145)

3 段構え。RN / Web 両対応で **Node crypto を使わず純 JS ハッシュ**。

1. **クライアント同期ヒューリスティック** `checkSpamBeforePost(content)` → `{ isSpam, reason }`:
   - 空(`isEmpty`)/ 同一文字 10 連続(`repeatChars`, `/(.)\1{9,}/u`)/ 20 字超で英大文字率 >80%(`allCaps`)/ URL 3 件以上(`tooManyLinks`)/ `CONTENT_FILTER_PATTERNS` 一致(`suspiciousPatterns`)。
   - `CONTENT_FILTER_PATTERNS`: 日本語スパム(「副業.*稼げる」「月収.*万円.*保証」「LINE.*追加.*プレゼント」等)+ フィッシング(「パスワード.*入力.*急」等)+ 記号連打(`[!！]{5,}`, `★{4,}`)。不正な正規表現は try/catch で無視。
2. **サーバーレートリミット** `checkRateLimitServer()` → RPC `check_and_log_post_rate`:
   - migration 0145 の制限: **10 分以内 5 投稿 / 1 時間以内 20 投稿**。超過は errcode **`53400`** で raise。`post_rate_log` に記録し、1 日以上前のログを自動削除。
   - クライアントは `53400` を「頻度高すぎ」メッセージに、未認証は `42501`。**ネットワーク断は fail-open**(`{ allowed:true }` でサーバー側 RLS に委ねる)。
3. **重複コンテンツ** `checkDuplicateContent(content)` → RPC `check_duplicate_content(p_content_hash)`:
   - クライアントは **djb2 ハッシュ**(`djb2Hash`, `codePointAt` でサロゲートペア対応, 8 桁 hex)を送る。
   - 返り値は `boolean` or `{ is_duplicate }`。エラー時 **fail-open で false**(投稿を止めない)。
- `runAllChecks(content)`: 1→2→3 の順。クライアントで弾けばサーバー呼ばない(早期 return)。
- **UI 統合**: `create.tsx` の投稿前チェックに組込み(grep で `runAllChecks` 等の参照あり)。

### 6. ユーザーブロック (匿名対応, migration 0143)

- **キーは `pseudonym_id`**(migration 0116 で導入。投稿ごとに一定期間同一ユーザーへ同じ値が割り当てられる匿名識別子)。実名ではなく仮名でブロックするのが匿名 SNS としての肝(→ [[匿名性設計と de-anon ホール]])。
- **DB** (`0143_user_blocks.sql`): `user_blocks(blocker_id, blocked_pseudonym_id, reason, created_at)`、PK = `(blocker_id, blocked_pseudonym_id)`。RLS `blocks_self_all`(`auth.uid() = blocker_id`)。`reason` は `'spam'|'harassment'|'other'`。
  - RPC は全て `security definer`: `block_pseudonym(p_pseudonym_id, p_reason)`(on conflict do nothing)/ `unblock_pseudonym` / `get_blocked_pseudonyms`(`returns table(blocked_pseudonym_id, reason, created_at)`, stable)。`authenticated` に grant。
- **Store** (`stores/blockStore.ts`):
  - `blockedPseudonymIds: Set<string>` で **O(1) lookup**。storage key `geek:blocks:v1`(Set を array にシリアライズ)。
  - **楽観更新**: `blockUser`/`unblockUser` はローカル即反映 → RPC は fire-and-forget。**通信失敗してもローカルを巻き戻さない**(fail-silent, swallow)。
  - `loadBlocks()`: RPC 未適用時の **`PGRST202` / `42883`(undefined_function)を捕捉してローカル状態維持**(silent skip)。返り値が `string[]` でも `{pseudonym_id}[]` でも吸収。
  - `isBlocked(pseudonymId)` / `clear()`(ログアウト時)。
- **API 薄ラッパー** (`lib/api/userBlocks.ts`): `blockUser` / `unblockUser` / `getBlockedUsers` → 同 3 RPC を `withApiTimeout` で。**store 版と違い throw する**(React Query mutation 用)。返り型 `BlockedUser { pseudonymId, reason, createdAt }`。
- **UI 統合** (`AnonPostCard.tsx`):
  - 「...」→ ブロック(`canBlock = !isOwnPost && !!post.pseudonym_id`, L790)。`blockUser(pid, 'harassment')` + **undo トースト**(`onUndo: () => unblockUser(pid)`)、Haptics.Warning(memory)。
  - L505–509: `pseudonym_id` は `/user/${pseudonymId}` ルーティングにも使う(de-anon Phase2)。

### 7–8. AI コンテンツ検出 scaffold (`lib/safety/contentFilter.ts`)

- ★ **Edge Function `moderate-content` は未実装(scaffold のみ)**。本実装時の設計コメントあり: Supabase Edge Functions + **Anthropic API (claude-haiku-4-5)** で ①分類 ②`post_moderation_queue` 保存 ③高リスクは自動非表示+管理者通知。`ANTHROPIC_API_KEY` は**クライアントバンドルに含めない**(Edge 側のみ, → [[認証・セッション]] / CLAUDE.md §8 の secrets 方針)。
- `reportContentForAIReview(postId, reportType)`: `supabase.functions.invoke('moderate-content')`。Edge 未デプロイ/通信エラーは **false で握り潰し**(UX を壊さず管理者が後対処)。`ContentReportType = 'spam'|'harmful_image'|'harassment'|'misinformation'|'other'`。
- `getModerationStatus(postId)`: `post_moderation_queue` を `maybeSingle`。**テーブル未作成(`PGRST116` / `42P01` / "relation ... does not exist")は graceful に null**。`ModerationStatus { status: 'pending'|'approved'|'rejected', reason? }`。

## 注意点・地雷

- **★ 重複検知のハッシュ不一致(機能していない疑い・要修正)**: クライアント `checkDuplicateContent` は **djb2** ハッシュ(`lib/safety/spamDetection.ts` の `djb2Hash`)を送るが、サーバー RPC `check_duplicate_content`(migration 0145)は **`md5(content)` と比較**している。アルゴリズムが違うので**永遠に一致せず重複検知が常に false** になっている可能性が高い。さらに RPC コメントは「`content_hash` カラムがあれば使う」と書きつつ実際は `md5(content)` をその場計算。**重複投稿防止は実質ノーガード**。修正するならクライアント側を `md5` に合わせるか、サーバーを djb2 受け入れに変える(どちらかに統一)。fail-open 設計なので投稿は通るが、機能としては死んでいる。
- **★ シェア URL ドメインのズレ**: `sharePost.ts` の `BASE_URL='https://geek-app.netlify.app'` に対し、CLAUDE.md / 運用ノートの Web 本番は **`geekboard.netlify.app`**。共有/埋め込みリンクが実ドメインを指していないと 404 になる。デプロイ前に実 URL と突合すること(→ [[運用 — デプロイ・プレビュー・本番反映確認]])。
- **クロスポスト選択 UI が無い**: API(`crossPost.ts`)はあるが呼ぶ画面が未実装。「機能がある」と誤認しないこと。
- **下書き一覧画面が未整備**: `?draftId=` で開く再開フローはあるが、ユーザーが下書きを一覧から選ぶ独立画面が残課題(memory 残タスク)。
- **ブロックは pseudonym_id ベース ≠ 永続ブロック**: `pseudonym_id` は「一定期間内は同一ユーザーへ同じ値」。期間が変われば値が変わり得るため、**長期的に同一ユーザーをブロックし続ける保証はない**。匿名性とのトレードオフ。実名 `author_id` でブロックすると de-anon になるため敢えて仮名キー(設計意図)。
- **フィードからブロック投稿を消す処理の所在**: `blockStore.isBlocked()` は提供されるが、フィード描画側で**実際に非表示にしているか**は別途確認推奨(store にあるだけでは消えない)。
- **migration 未適用時の silent degrade**: blockStore(`PGRST202`/`42883`)・contentFilter(`42P01`)・spam(fail-open)はいずれも**未適用でもエラーを出さず素通り**する。0142–0145 は memory 上「適用済み」だが、別環境では**機能が静かに無効化される**ので適用確認が必要(同種の罠は [[Admin Console (運営管理)]] の 0118–0123 でも既出)。
- **fire-and-forget 同期の整合性**: 下書き/ブロックはローカル先行 + サーバー非同期。通信失敗でローカルとサーバーが乖離する(ローカルは巻き戻さない設計)。多端末で挙動が分かれる可能性。
- **引用の FlashList dedup はモジュールキャッシュ**: `_quoteCache` はセッション内永続(画面遷移で消えない)。引用元が編集/削除されてもキャッシュが古いまま残り得る(削除は `on delete set null` で `quote_post_id` 自体が消えるので実害は限定的)。
- **全機能 `swallow` で握り潰す**: デバッグ時は Sentry breadcrumb(`blockStore.blockUser` 等の固定 scope)で追う。`contentFilter` のみ `console.warn` 直書き(他は swallow)。
