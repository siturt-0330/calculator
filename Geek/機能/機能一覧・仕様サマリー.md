## 概要

### GEEK とは
**GEEK(ギーク)** = 日本の「オタク文化 / 推し活 / 趣味コミュニティ」専用の **完全匿名型 SNS**。
iOS / Android / Web を **単一コードベース**(React Native + Expo)で配信。バックエンドは Supabase。

- アプリ名: GEEK / バージョン: 4.0.0
- Web 本番: <https://geekboard.netlify.app/>
- アクセントカラー: 紫 `#7C6AF7`
- 料金: Free / Pro の 2 プラン(★Pro は未リリース・将来予定)
- 開発体制: **開発者 1 人 + Claude ペアプロ**(Lean Startup 的に小さく検証→捨てる)

### 解決する課題(プロダクトの WHY)
| 既存 SNS の問題 | GEEK の解決策 |
|---|---|
| 実名・知人紐付けで本音が言えない | 全投稿が **匿名表示**(コメント欄は「1番目の人」番号) |
| 炎上・誹謗中傷リスク | **信頼スコア** + 投稿前 **AI 検査** + **タグフィルター**で構造的に抑制 |
| TL がノイズだらけ | **タグ中心設計**(フォローでなくタグで繋ぐ)。ブロックタグで完全除外 |
| 公式情報と噂の混在 | **公式コミュニティ制度**(URL 所有権検証 + 運営審査で認証) |
| コミュニティ分散 | タグ・コミュニティ機能で同好の士を 1 か所に集約 |

### コアバリュー 4 つ
1. **完全匿名性** — 投稿者の実名・アカウントは絶対に表示しない
2. **構造的安全性** — ルール禁止でなく「仕組みとして起きにくく」(信頼スコア / AI 検査 / 気になる / タグフィルタ)
3. **タグ中心の情報設計** — インフルエンサーによる情報独占を防ぐ
4. **プライバシー尊重** — 広告は個人 ID 非送信(タグ情報のみ)/ ローカル学習はデバイス内 / Sentry は PII redact

詳細な匿名性の設計と既知ホールは [[匿名性設計と de-anon ホール]] を必ず参照。

## 仕組み・設計(機能カテゴリ別 + 具体ファイルパス)

> ファイルパスは `geek-v4/` 起点。Expo Router の file-based routing なので `app/` 配下のパス = URL。

### A. 認証・オンボーディング
- **認証方式**: Supabase Auth (PKCE)。session key = `geek-v4-auth`。**SecureStore は使わず** localStorage(web)/ AsyncStorage(native)。client は `lib/supabase.ts`。
- **画面**: `app/(auth)/`(login / signup)、`app/onboarding/`(6 画面ウィザード: ウェルカム→言語→ニックネーム→好きなタグ→ブロックタグ→通知)。
- **アカウント状態 5 段階**: healthy / caution / restricted / warned / suspended。`refresh_account_state` トリガが懸念/投稿比率(0.5→caution, 1.0→restricted, 1.5→warned)で自動算出。suspended のみログイン不可。
- セッションの根本挙動・「急に login に戻る」事故は [[認証・セッション]] へ。

### B. メインナビゲーション
- 浮遊型ダークピル(Slack 風)タブバー。`app/(tabs)/` に **5 タブ**: ホーム(feed)/ 掲示板(bbs)/ コミュニティ / マイページ + 中央 FAB(投稿作成)。
- community は tab bar 保持のため `app/(tabs)/community/` 配下に置く設計。

### C. フィード(ホーム)
- 画面: `app/(tabs)/feed`。投稿カードは **`components/feed/AnonPostCard`**(分割済み)。
- **スコープ**: All(全タグ。ブロックタグ除外)/ For Me(好きなタグのみ。未設定だとグレーアウト)。
- **ソート**: おすすめ(For You)/ 急上昇(Hot)/ 新着(New)/ 評価順(Top)。
- **ページネーション**: 1 回 40 件のカーソル方式(いいね数 + 投稿時刻の複合カーソル)。
- **広告**: 8 投稿ごとに 1 件挿入。ターゲティングはタグのみ・個人 ID 非送信。パーソナライズ広告 OFF で広告自体が消える。
- **RPC 集約**: `supabase/migrations/0041_get_feed_page_rpc.sql`(周辺データを 1 RPC で)→ `lib/api/feedPage.ts` / `hooks/useFeedPage.ts`。home feed 1 ページ目集約 RPC は `0114`(flag `EXPO_PUBLIC_HOME_FEED_RPC=1`、★既定 OFF)。
- **For You 個人化(Value Model)**: `lib/personalize/`(score.ts / events.ts / profile.ts + syncAffinity.ts / impressions.ts)+ migration `0139`(タグ親和性)`0140`(impression)`0141`(`get_for_you_feed` RPC)。flag `EXPO_PUBLIC_FOR_YOU_FEED_RPC=1`(★既定 OFF / 0139-0141 適用 + pg_cron 登録後に有効化)。
- **realtime**: feed 全体は `hooks/useFeedRealtime.ts` に集約。cache 書き戻しは `lib/cacheUpdates/feedPagePatcher.ts`。
- ランキング詳細は [[フィード・ランキング・レコメンド]]、cache パターンは [[State管理 (Zustand・React Query)]]。

### D. 投稿作成
- 画面: `app/post/create.tsx`(modal)。本文 **最大 2000 字** + 画像 **最大 4 枚** + タグ **1〜5 個**。
- 種別バッジ: fact / opinion / joke / wip。CW: spoiler / nsfw / violence / sensitive(+説明 120 字)。
- 公開範囲: 自分だけ / 一般公開 / コミュニティメンバーのみ / コミュニティ公開。
- **投票(ポール)**: 選択肢最大 6 / 複数選択トグル / 期間 6・24・72・168h。
- **下書き自動保存**: `stores/draftStore.ts`(Zustand + MMKV, 最大 20 件, 30 日 TTL, サーバー同期 `0144_post_drafts.sql`)。create.tsx は 1.5s debounce で「下書き保存済み」表示。
- **投稿前 AI 検査**: Edge Function `check-content`(個人情報・誹謗中傷を正規表現 + NFKC で検査)。**fail-secure**(catch で `ok:false`)。
  - ⚠️ 仕様書本文 §18-3 は「fail-open」と書くが、**CLAUDE.md / Edge Function は fail-secure(catch で ok:false)**。コードを正とする。
- 引用投稿 / クロスポスト / シェア等の追加機能は [[プラットフォーム機能 (引用・シェア・下書き・ブロック)]]。

### E. 投稿詳細・コメント
- 画面: `app/post/[id].tsx`。「似た投稿」最大 6 件(共通タグ数 + 意味的類似)。
- コメント最大 500 字 / 番号(#1, #2…)+ ランダムカラーアバター + 信頼バッジ。realtime 自動追加。

### F. 匿名掲示板(BBS)
- 画面: `app/bbs/[id].tsx`(詳細)/ `app/bbs/create.tsx`(作成)。スレッド + 返信形式(5ch 的)。
- 12 カテゴリ(雑談 / アニメ / ゲーム / マンガ / 音楽 / アイドル / Vtuber / 推し活 / グルメ / コスプレ / ニュース)。
- **検索はクライアント側**(`lib/search/`): parseQuery(キーワード / "フレーズ" / -除外)→ generateVariants(表記揺れ。`MAX_VARIANTS=24` cap)→ deepNormalize → BM25 ライク scoring + CTR ブースト → 0 件時 findClosestK(レーベンシュタイン「もしかして」最大 3)。
- 返信最大 500 字 / 引用 `>>N` / `@` メンション補完。**投稿に信頼スコア 10 以上が必要**。

### G. コミュニティ
- 画面: `app/(tabs)/community/`、公式管理は `app/official/`(admin 限定)。
- 種類: 一般(オープン 🌐 / 許可制 🔒 / 完全招待制 🛡️)と **公式**(運営認証・青バッジ)。
- タブ(遅延ロード + Keep-Alive): ホーム / 掲示板 / 聖地(マップ)/ カレンダー /(公式は)Q&A・コメント。
- 参加は **楽観的更新**(応答前に「参加中」表示、エラーで revert)。
- アイコン upload は EXIF strip + magic-byte 検証 + JPEG 再エンコード(`lib/image.ts`)。bucket = `community-icons`。
- **聖地(スポット)**: React Native Maps。一般メンバー候補投稿 → 管理者認証。
- **カレンダー / イベント**: 月別グループ化。`maybe_promote_proposal` トリガ(提案が 10% 同意で公式イベント昇格)。
- **Q&A(公式限定)**: RAG。管理者がナレッジ登録(`qna_documents`)→ AI が参照回答 + 出典表示。
- **匿名剥奪返信**: 公式管理者は特定ユーザーの匿名を解除して返信(`official_author` 表示)。

### H. マイページ / 設定
- 画面: `app/(tabs)/mypage`、サブ `app/mypage/{saved,liked,posts}.tsx`、`app/settings/`。
- KPI: 投稿数 / もらった♥ / 参加コミュ数。アクティビティ(自分の投稿 / いいね / 保存 / 運営メッセージ)。
- プライバシー設定: 気になるをプライベート化 / データセーバー / モーション削減 / パーソナライズ広告 OFF。
- データエクスポート + アカウント削除(猶予期間あり・原則取消不可)。

### I. 通知
- 7 種: like / comment / reply / follow / event / official_post / announcement。一覧 `app/notifications/`(最新 50 件、開いた時点で既読化)。
- **Web Push**: VAPID + Edge Function `send-push`(失効 endpoint 404/410 を自動削除)。おやすみ時間(日跨ぎ対応)。

### J. 検索(横断)
- 画面: `app/search.tsx`。post / bbs / tag / community / user を横断。
- 表記揺れ・タイポ補正・検索履歴(最大 50)・**Learning to Rank**(クリック履歴をデバイス内記録 → 再検索で上位化)。

### K. タグシステム
- 画面: `app/filter/index.tsx`(タグフィルタ modal)。通常タグ / 公式タグ / 階層タグ(親子)/ シノニム(投票承認)。
- **タグ共起マトリクス**: `lib/tagClustering/`(hub-based cluster + co-occurrence)。TTL 1h・最新 500 投稿・デバイス内キャッシュ。
- **ゴシップ/スキャンダル/事件報道系タグは、ユーザー未設定でもトレンドに絶対表示しない**(プロダクトポリシー)。`tests/unit/trending.test.ts`。

### L. 信頼スコア(trust score) ★実装は仕様書と別物
- 計算: **`lib/trust/score.ts` の純関数**(profile 行だけで算出、DB トリガ追加なし)。
- **実際の式(2026-05 改修済)**: `score = clamp(round(BASE + posts + likes + comments + active_days − concerns), 0, 100)`
  - BASE = **30**(全員付与)
  - posts: +0.5/件(cap 15) / likes: +1.0/件(cap 25) / comments: +0.4/件(cap 10) / active_days: +0.3/日(cap 12) / **concerns: −3/件(cap −30)** ← 減点項あり
- **ティアは内部判定のみ**(UI には数値スコア 0-100 だけ表示。`app/settings/trust-score.tsx`)。tier key = `newcomer`(0-29)/`regular`(30-69)/`probably_nice`(70-89)/`definitely_nice`(90-99)/`god`(100)。肩書(新参者/常連/神)と emoji/perks は休眠コード化したため撤去済み。
- 投稿制限: **BBS は 10 以上 / フィードは 20 以上**(spec §16-2)。
- 「気になる(concern)」= いいねの逆。理由: misinfo / unverified / spam / rude / scam / other。
- 計算の別経路: Edge Function `calculate-trust-score`(サーバー側再計算)。
- ⚠️ **3 ソースで値が全部違う**(下記「注意点・地雷」参照)。**`lib/trust/score.ts` を正とする**。

### M. デザインシステム / i18n / テーマ
- トークン: `design/tokens.ts`(C / GRAD / SP / R / SIZE)+ typography / shadows / motion / haptics。
- ⚠️ **フォントの食い違い**: spec §19-2 は「見出し=Syne / Inter / Noto Sans JP」、ロゴ=Orbitron Black と書くが、**CLAUDE.md(イントロ確定版)はロゴ = Apple system stack(`LOGO_FONT`, weight 800, 46px)**。ロゴ演出は変更禁止の確定版なので CLAUDE.md / `design/typography.ts` を正とする。
- i18n: 静的は `lib/i18n.ts` の `DICT` + `useT()`。動的(投稿本文)は `translateDynamic()`(MyMemory API)。**ブランド名 "Geek" は protectBrandNames で翻訳から保護**。9 言語: ja / en / zh / ko / es / fr / th / vi / id。
- 翻訳 UI(バッジ・ボタン)は撤去済み。翻訳は **サイレント**動作のみ。
- 詳細は [[i18n・テーマ・デザインシステム]]、画像周りは [[画像・メディアパイプライン]]。

## 注意点・地雷

### ⚠️ 仕様書(SPECIFICATION.md)とコードの乖離(最重要)
仕様書は **2026-05-22 で凍結**しており、その後の実装に追随していない。鵜呑み禁止。

| 項目 | SPECIFICATION.md の記述 | 実コード / CLAUDE.md(=正) |
|---|---|---|
| **DB マイグレーション数** | 「0035 まで」(§25-4) | **0145 まで(144 ファイル)+ complete_schema.sql** 実在 |
| **信頼スコア式** | 年齢20+投稿30+いいね30+コメ20、減点なし(§16-1) | **BASE 30 + posts/likes/comments/active_days − concerns**(`lib/trust/score.ts`) |
| **信頼ティア** | 4 段階 0-19/20-49/50-79/80-100(§11-10)。※本文 §16 と memory はさらに別値 | **5 段階** newcomer 0-29 / regular 30-69 / probably_nice 70-89 / definitely_nice 90-99 / god 100。**UI には数値のみ表示**(肩書/emoji は撤去) |
| **AI 検査の失敗時挙動** | fail-open(§18-3) | **fail-secure**(catch で `ok:false`) |
| **ロゴフォント** | Orbitron Black(§19-2 / §6-2) | **Apple system stack**(`LOGO_FONT` weight 800, 46px)。イントロ確定版で変更禁止 |
| **アプリ構成のディレクトリ** | `app/profile/` を記載(§25-2) | 実際は `app/(tabs)/mypage` + `app/mypage/*`。`app/admin/`(隠し)も存在 |

→ memory `project_geek_v4_spec.md` も古い(ティア値 0-25/26-50/… は **どのソースとも一致しない誤り**)。**`lib/trust/score.ts` が唯一の真実**。

### ⚠️ Supabase 手動適用が必要な migration(silent degrade の罠)
master に merge しても **Supabase SQL エディタで手動適用しないと効かない**ものがある。未適用でも fallback で「動いてしまう」ため気付きにくい:
- Admin Console: `0118`〜`0123`(→ [[Admin Console (運営管理)]])
- Value Model 個人化: `0139`〜`0141`(+ pg_cron 登録 + flag)
- ★ posts 列改竄ホール封じ `0134` / author_id revoke 是正 `0138` は **適用前に like→count を実機確認必須**(memory `project_geek_v4_post_column_hole.md`)。
- 機能フラグ既定 OFF(`EXPO_PUBLIC_HOME_FEED_RPC` / `EXPO_PUBLIC_FOR_YOU_FEED_RPC`)は **`=== '1'` で判定**。`!== '0'`(既定 ON)パターンをコピペすると事故。

### ⚠️ Realtime の publication 未登録 table
`concerns` / `saves` / `community_stamp_reactions` は publication 未登録 → **subscribe するだけで CHANNEL_ERROR**。1 channel に複数 table を chain すると 1 つの error で全死。必ず `attachChannel` を table 別に。詳細は [[Realtime]]。

### ⚠️ 画像表示 `resize=cover` 潰れ
`thumbedUrl()`(`lib/utils/imageUrl.ts`)に **width だけ + `resize=cover`** を渡すと幅だけ縮小し高さ据え置きで画像が潰れる。height 未指定は `resize=contain` 既定(壊すな)。「写真が縦長/潰れる」を見ても EXIF を疑う前にこれ。直近コミット `b138c51` で関連修正。→ [[画像・メディアパイプライン]] / [[地雷・落とし穴 総覧]]。

- ロードマップ(§27)の DM / ライブ配信 / AI 本格統合(現状 suggest-caption はスタブ)/ 海外展開は **未実装の予定**。AI 検出(`moderate-content` Edge Function)も scaffold のみ。
